# Wave 4.1 — App Store Connect + RevenueCat Setup Runbook

This is the founder's step-by-step for provisioning Wayfind Pro on
App Store Connect and RevenueCat. Everything below is dashboard work;
the SDK wiring lives in Wave 4.2.

> **Sequencing matters.** Complete §1 before §2 — RevenueCat imports
> products from App Store Connect. Complete §2 before shipping Wave
> 4.2 to TestFlight — the iOS SDK fails to configure cleanly without
> a published RevenueCat offering.

---

## 1. App Store Connect

### 1.1 Subscription Group

Create one group: **`wayfind_pro`**.

| Field | Value |
|---|---|
| Reference Name | `wayfind_pro` |
| Localized Name (en-US) | `Wayfind Pro` |
| Family Sharing | **Enabled** |

> Family Sharing on by default — the cost is negligible (one
> entitlement per Apple ID family, not per device) and removing it
> later is treated as a price-up by Apple, requiring user re-consent.

### 1.2 Subscription Products

Create three auto-renewing subscriptions in the group:

| Product ID | Display Name | Duration | Price (USD Tier) | Notes |
|---|---|---|---|---|
| `wayfind_pro_monthly` | Wayfind Pro · Monthly | 1 month | $4.99 (Tier 5) | Default purchase |
| `wayfind_pro_annual` | Wayfind Pro · Annual | 1 year | $39.99 (Tier 40) | Headline value (33% off vs. monthly) |
| `wayfind_pro_founder_annual` | Wayfind Pro · Founder Annual | 1 year | $19.99 (Tier 20) | Promo offer only (see §1.5) |

Each product:

* **Family Sharing**: enabled.
* **Subscription Group**: `wayfind_pro`.
* **Localizations**: en-US first; ship pt-BR / es-ES in a follow-up.
* **App Store promoted**: enable for monthly + annual (lets Apple
  feature them in editorial picks). Skip for the founder SKU.

### 1.3 Introductory Offer

| Type | Length | Price | Eligibility |
|---|---|---|---|
| Free Trial | 7 days | $0 | First-time subscribers in `wayfind_pro` |

Apply the same intro offer to all three product IDs so users who
upgrade from monthly to annual mid-trial don't lose remaining days.

### 1.4 App Review Notes

Add the following to the App Review Information panel — Apple
historically rejects subscription apps without an explicit "what's
free vs. what's paid" breakdown.

> Wayfind is a free trip planner. Wayfind Pro unlocks:
>
> * Unlimited AI day plans (free tier: 3/month).
> * Unlimited trip documents (free: 5/trip).
> * CSV expense export.
> * Multi-currency budget conversion.
> * Live flight status tracking with push notifications.
>
> No content is gated server-side; existing user data remains
> accessible without a subscription. Restore Purchases is in
> Settings → Subscription. The app honors Family Sharing.
>
> Test account for App Review: `appreview@wayfind.app` / password
> `<filled at submission time>`.

### 1.5 Promo Offers (Founder Annual)

Set up `wayfind_pro_founder_annual` as a **promo code recipient**:

1. In App Store Connect → Subscriptions, open the Founder Annual SKU.
2. Add a **Promo Offer** (not Intro Offer):
   - Reference name: `founder_2026`.
   - Offer code key: generated by Apple — store in 1Password under
     `wayfind/founder_offer_key`.
3. Generate the offer codes via App Store Connect → Promo Codes
   in batches of 100 as marketing demands.

The iOS client doesn't need any changes for promo codes — Apple's
StoreKit 2 surface handles redemption directly via deep links. Wave
4.6 surfaces the "Redeem code" button in Settings.

### 1.6 Tax / Bank Setup

| Step | Owner | Status |
|---|---|---|
| Bank account validated | Founder | ☐ |
| Tax forms (W-9 + W-8BEN if applicable) | Founder | ☐ |
| Paid Apps Agreement signed | Founder | ☐ |

Apple holds revenue until all three are checked. Confirm before
running marketing.

## 2. RevenueCat

### 2.1 Project Setup

1. Create RC project: `wayfind`.
2. Add an iOS app: bundle id `com.wayfind.ios` (production) and
   `com.wayfind.ios.dev` (TestFlight). Tie both to the same project so
   sandbox + production share entitlements.
3. Upload the App Store Connect API key (Issuer ID + Key ID + .p8 from
   ASC → Users and Access → Keys → App Store Connect API). RC uses
   this to validate receipts server-side.

### 2.2 Entitlement

Create one entitlement: **`wayfind_pro`**. This is the string the iOS
client checks via `purchases.customerInfo.entitlements["wayfind_pro"].isActive`.

### 2.3 Offerings

Create one offering: **`default`** (mark as Current Offering). Add
three packages:

| Package Identifier | Product | Notes |
|---|---|---|
| `$rc_monthly` | `wayfind_pro_monthly` | Standard package id — let RC's no-code paywall recognise it |
| `$rc_annual` | `wayfind_pro_annual` | Standard package id |
| `wayfind_founder_annual` | `wayfind_pro_founder_annual` | Custom id — only surfaced via deep link |

### 2.4 Paywall (No-Code Editor)

Use the **Paywall Editor** (RevenueCat dashboard → Paywalls → New
Paywall → "Use template").

| Field | Value |
|---|---|
| Headline | **Unlimited AI day plans** |
| Sub-headline | "Plus live flight tracking, trip docs, CSV export, and multi-currency conversion." |
| Primary CTA | "Start 7-day free trial" |
| Secondary CTA | "Restore purchases" |
| Default selected package | `$rc_annual` (drives ARPU) |
| Footer | "Cancel anytime in Settings. Family Sharing supported." |

### 2.5 Webhook

Point RevenueCat → Integrations → Webhooks to:

```
POST https://<supabase-project>.functions.supabase.co/revenuecat-webhook
Header: X-RevenueCat-Signature (HMAC; secret in RC dashboard)
```

The `revenuecat-webhook` Edge Function is already deployed (see
Wave 4.4 for the hardening pass that adds REFUND + SUBSCRIBER_ALIAS
handlers + idempotency).

### 2.6 Sandbox Test Accounts

Create at least three sandbox testers in App Store Connect → Users
and Access → Sandbox Testers:

| Use case | Email pattern |
|---|---|
| Happy-path purchase | `wf+rc-buyer@<domain>.com` |
| Trial → cancel | `wf+rc-trialcancel@<domain>.com` |
| Ask-to-Buy (Family Sharing child) | `wf+rc-askbuy@<domain>.com` |

Document credentials in 1Password under `wayfind/sandbox`.

### 2.7 A/B Test (Day-One)

Create a 50/50 split test on the paywall:

| Variant | Difference |
|---|---|
| `default` | Headline "Unlimited AI day plans" |
| `flight_first` | Headline "Live flight alerts. Free 7 days." |

Run for 14 days minimum so we have ≥ 100 conversions per variant
before declaring a winner. RC handles randomisation server-side.

## 4. Pricing Rationale

* **$4.99/mo, $39.99/yr** sits 30% below TripIt Pro ($49) and matches
  Wanderlog Pro ($45) — comfortable mid-market without triggering
  expectations of premium-tier features.
* **$39.99 annual** = 33% savings vs monthly; tracks the
  consumer-app norm and makes the annual the obvious "right choice"
  in the paywall.
* **$19.99 Founder Annual** is a 50% discount tied to a finite
  promo-code run (1,000 codes). Used as a HackerNews / Twitter
  growth lever for the first 30 days. After the codes are gone the
  SKU stays in App Store Connect as a "we honored your purchase"
  artifact for renewals.
